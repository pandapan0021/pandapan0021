descriptor与property

Veröffentlicht am 2017-03-24

descriptor

描述器是指作为对象的属性存在并且定义了__get__,__set__,__delete__方法的对象。
描述器改变了对象属性的默认访问方式。
当访问对象属性时，默认的行为方式如下：
a.x -> a.__dict__[‘x’] -> type(a).__dict__[‘x’]…..

当说访问的属性定义了描述器方法的时候，则以如下方式访问：
a.x -> x.__get__(a, type(a))
a.x = 1 -> x.__set__(a, 1)
del a.x -> x.__delete__(a)

当通过类对象而不是实例对象访问时描述器方法的instance参数对None
当对象定义了__set__/__delete__中的任意一个或都定义时，此对象为一个数据描述器，数据描述器总是覆盖实例中重新定义的同名属性。如果对象只定义了__get__方法，则它为一个无数据描述器，实例中定义的属性可以覆盖此描述器。

descriptor的应用

1.staticmethod()

def staticmethod(f):
    class wrap(object):
        def __get__(self, instance, owner):
            return f
    return wrap()

这样就实现了一个简单的静态方法包裹函数。

3.property

property函数有两种使用方法
方法一、

class C:
    def __init__(self):
        self._x = None
    def getx(self):
        return self._x
    def setx(self, value):
        self._x = value
    def delx(self):
        del self._x
    x = property(getx, setx, delx, "x property")

方法二、

class C:
    def __init__(self):
        self._x = None
    @property
    def x(self):
        return self._x
    @x.setter
    def x(self, value):
        self._x = value
    @x.deleter
    def x(self):
        del self._x
class property(fget=None, fset=None, fdel=None, doc=None)简单实现
class property(object):
    def __init__(self, fget=None, fset=None, \
                       fdel=None, doc=None):
        self._fget = fget
        self._fset = fset
        self._fdel = fdel
        self.__doc__ = doc
    def setter(self, f):
        self._fset = f
        return self
    def deleter(self, f):
        self._fdel = f
        return self
    def __get__(self, instance, owner):
        return self._fget(instance)
    def __set__(self, instance, value):
        self._fset(instance, value)
    def __delete__(self, instance):
        self._fdel(instance)
